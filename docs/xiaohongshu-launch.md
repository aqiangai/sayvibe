# say vibe 小红书宣发方案

这份文档基于当前仓库主页与现有截图整理，目标不是写“项目说明书”，而是把 `say vibe` 转成适合小红书发布的内容。

## 账号定位

推荐把 `say vibe` 放在下面这个叙事里：

- 不是“做了一个工具”
- 而是“我把 iPhone 输入法的语音能力，接到了 Mac 的真实输入工作流”
- 核心感受是：更安静、更低摩擦、更容易进入心流

更适合你的小红书表达：

- 安静输入
- 小声 vibe
- 不用额外硬件
- 方言友好
- 会议纪要顺手沉淀
- iPhone + Mac 工作流

## 发布原则

### 第一条先发什么

第一条不要讲太多技术栈，也不要直接把 README 原文搬过去。

第一条最适合发：

- 真实场景
- 为什么做它
- 它具体解决了什么问题
- 看起来到底是什么样

### 第一条先不要发什么

建议首发先不要把“个人微信 / 群二维码 / 打赏二维码”直接放在正文主内容里。

更稳的方式：

- 首发先跑自然互动
- 评论区先承接“想试试 / 想看教程 / 想进群”
- 第二轮或主页资料里再慢慢承接社区入口

## 素材顺序

推荐首发使用 4 张内容：

1. `docs/media/mobile-vibe.jpg`
   说明：第一眼让人知道这是手机端的真实使用场景
2. `docs/media/desktop-console.png`
   说明：证明不是概念图，Mac 端确实已经联动起来了
3. `docs/media/mobile-settings.jpg`
   说明：展示扫码 / 局域网连接 / 自动输出动作
4. `docs/media/vibe-demo.gif`
   说明：放最后做动态收尾，更像“真实工作流”

如果小红书发图文不方便带 GIF，就把 GIF 单独留到第二条，做“实际操作演示”。

## 封面文案建议

封面尽量只放一句话，不要太满。

可选 1：

```text
把 iPhone 语音输入
接到 Mac 上
```

可选 2：

```text
我做了一个
安静输入工具
```

可选 3：

```text
小声说话
也能快速落字
```

默认推荐：

```text
把 iPhone 语音输入
接到 Mac 上
```

原因：

- 一眼就能看懂是 `iPhone + Mac` 联动
- 设备词和需求词都在
- 比“安静输入工具”更适合搜索和停留

## 标题策略

第一条最重要的不是“文艺感”，而是让用户一眼知道这是什么。

优先级应该是：

1. 设备词要清楚
2. 需求词要清楚
3. 结果词要短

更适合这条笔记的标题结构：

```text
[设备词] + [动作词] + [结果词]
```

例如：

```text
把iPhone语音接到Mac
```

## 首发标题备选

控制在 20 字以内，优先这些：

1. 把iPhone语音接到Mac
2. iPhone语音直接进Mac
3. 不用硬件的语音输入流
4. 小声vibe也能快速打字
5. 我做了个安静输入工具

我最推荐：

```text
把iPhone语音接到Mac
```

标题说明：

- `把iPhone语音接到Mac`
  - 最适合首发，信息密度最高，搜索意图最明确
- `iPhone语音直接进Mac`
  - 更口语，适合第二次发或复发测试
- `不用硬件的语音输入流`
  - 更偏“方法论”，适合第二条内容
- `小声vibe也能快速打字`
  - 更偏情绪表达，适合有一定粉丝基础后再用
- `我做了个安静输入工具`
  - 偏独立开发叙事，不如前两个直接

不建议首发用这些词：

- AI神器
- 黑科技
- 宝藏工具
- 效率神物

原因：

- 太泛
- 容易吸来不精准流量
- 会削弱 `iPhone / Mac / 语音输入` 这些真正该抢的词

## 首发正文

下面这版控制在小红书可用长度内，语气偏真实用户，不像项目文档。

```text
最近我一直在折腾一件事：
能不能直接用 iPhone 输入法的语音输入，
把话安静地说出来，
然后稳定落到 Mac 的输入框里。

我最后把它做成了一个小工具，叫 say vibe。

它对我最有用的不是“语音转文字”这四个字，
而是输入阻力真的小了很多。

有些时候你脑子里已经有想法了，
但手打很慢，或者声音不想太大，
节奏就会断掉。

现在我会直接拿起手机，
用系统输入法语音先把内容说出来，
它会同步到 Mac，
我再继续补改、整理、发送。

我自己最喜欢它的两个点：

1. 不用额外硬件，手机就能直接用
2. 直接复用输入法语音能力，普通话、英文、方言都更自然

我还顺手做了会议纪要模式，
输入完能直接当文档存起来，后面再导出。

现在已经把 iOS 和桌面端整理出来准备开源了。

如果你也对这种 iPhone + Mac 的输入工作流感兴趣，
我可以继续发：
连接方式、实际使用视频、会议纪要怎么用、以及开源进度。
```

先不要堆很多，保持 5 到 7 个就够了。

默认推荐：

```text
#iPhone技巧 #Mac技巧 #语音输入 #效率工具 #工作流 #独立开发
```

推荐备选 1：

```text

```

推荐备选 2：

```text
#语音输入 #效率工具 #iPhone技巧 #Mac技巧 #工作流 #独立开发
```

推荐备选 3：

```text
#iPhone技巧 #Mac技巧 #语音输入 #工作流 #效率工具 #会议纪要
```

标签策略说明：

- `#iPhone技巧`
  - 抢设备侧搜索
- `#Mac技巧`
  - 抢电脑侧搜索
- `#语音输入`
  - 抢真实需求词
- `#效率工具`
  - 承接泛效率流量
- `#工作流`
  - 承接高意图人群
- `#独立开发`
  - 给你个人叙事加身份标签

首发先不要上这些：

- `#AI工具`
- `#神器`
- `#宝藏工具`
- `#数码好物`

原因：

- 会把流量拉散
- 第一条更需要“精准的人”，不是“泛流量”
- `AI工具` 会让内容误入很拥挤的泛 AI 池子
