# say vibe 小红书宣发方案

这份文档基于当前仓库主页与现有截图整理，目标不是写“项目说明书”，而是把 `say vibe` 转成适合小红书发布的内容。

## 账号定位

推荐把 `say vibe` 放在下面这个叙事里：

- 不是“做了一个工具”
- 而是“我把 iPhone 输入法的语音能力，接到了 Mac 的真实输入工作流”
- 核心感受是：更安静、更低摩擦、更容易进入心流

更适合你的小红书表达：

- 安静输入
- 小声 vibe
- 不用额外硬件
- 方言友好
- 会议纪要顺手沉淀
- iPhone + Mac 工作流

## 发布原则

### 第一条先发什么

第一条不要讲太多技术栈，也不要直接把 README 原文搬过去。

第一条最适合发：

- 真实场景
- 为什么做它
- 它具体解决了什么问题
- 看起来到底是什么样

### 第一条先不要发什么

建议首发先不要把“个人微信 / 群二维码 / 打赏二维码”直接放在正文主内容里。

更稳的方式：

- 首发先跑自然互动
- 评论区先承接“想试试 / 想看教程 / 想进群”
- 第二轮或主页资料里再慢慢承接社区入口

## 素材顺序

推荐首发使用 4 张内容：

1. `docs/media/mobile-vibe.jpg`
   说明：第一眼让人知道这是手机端的真实使用场景
2. `docs/media/desktop-console.png`
   说明：证明不是概念图，Mac 端确实已经联动起来了
3. `docs/media/mobile-settings.jpg`
   说明：展示扫码 / 局域网连接 / 自动输出动作
4. `docs/media/vibe-demo.gif`
   说明：放最后做动态收尾，更像“真实工作流”

如果小红书发图文不方便带 GIF，就把 GIF 单独留到第二条，做“实际操作演示”。

## 标题策略

第一条最重要的不是“文艺感”，而是让用户一眼知道这是什么。

优先级应该是：

1. 设备词要清楚
2. 需求词要清楚
3. 结果词要短

更适合这条笔记的标题结构：

```text
[设备词] + [动作词] + [结果词]
```

你这次的新方向里，原句是：

```text
利用手机 vibecoding，不再使用三方大模型和大疆麦克风
```

这句话更适合：

- 封面文案
- 正文开头第一句
- 评论区延展表达

不建议直接当发布标题的原因：

- 太长
- 搜索词分散
- 标题位更适合放“设备词 + 动作词 + 结果词”

## 首发正文

下面这版控制在小红书可用长度内，语气偏真实用户，不像项目文档。

```text
最近我一直在折腾一件事：
能不能直接用 iPhone 输入法的语音输入，
把话安静地说出来，
然后稳定落到 Mac 的输入框里。

我最后把它做成了一个小工具，叫 say vibe。

它对我最有用的不是“语音转文字”这四个字，
而是输入阻力真的小了很多。

有些时候你脑子里已经有想法了，
但手打很慢，或者声音不想太大，
节奏就会断掉。

现在我会直接拿起手机，
用系统输入法语音先把内容说出来，
它会同步到 Mac，
我再继续补改、整理、发送。

我自己最喜欢它的两个点：

1. 不用额外硬件，手机就能直接用
2. 直接复用输入法语音能力，普通话、英文、方言都更自然

我还顺手做了会议纪要模式，
输入完能直接当文档存起来，后面再导出。

现在已经把 iOS 和桌面端整理出来准备开源了。

如果你也对这种 iPhone + Mac 的输入工作流感兴趣，
我可以继续发：
连接方式、实际使用视频、会议纪要怎么用、以及开源进度。
```

## 首发正文观点版

如果你这条就是要明确打“替代大模型 / 替代大疆麦克风”的观点，建议直接用这版：

```text
我最近在折腾一件事：
利用手机 vibecoding，
尽量不再依赖三方大模型，
也不再为了输入这件事专门上大疆麦克风。

我现在更常用的方式，
是直接拿起手机，
用豆包输入法这类系统输入法语音先把内容说出来，
然后同步到电脑继续补改。

它最打动我的点不是“AI 很强”，
而是整个输入动作变轻了很多。

很多时候不是没想法，
而是你不想开一堆模型界面，
也不想再额外挂一套硬件，
结果真正卡住的是开始输入这一步。

现在这套方式对我来说更自然：

1. 手机随手就能开始
2. 直接复用输入法语音能力
3. 普通话、英文、方言切换更顺
4. 到电脑上还能继续改，不会被一次性生成绑死

所以我现在更愿意把它理解成：
不是“让 AI 替我写”，
而是“先把想法更快地落下来”。

如果你也在折腾 vibe coding、
豆包输入法、
或者想摆脱额外硬件输入流，
我可以继续发实际演示。
```

## 标签建议

先不要堆很多，保持 5 到 7 个就够了。

默认推荐：

```text

```

```

```

```text

```

标签策略说明：

- `#大疆`
  - 吃品牌对比讨论流量
- `#typeless`
  - 给你这条内容留圈层识别度
- `#vibe`
  - 保留情绪和方式标签
- `#vibecoding`
  - 直接贴主题
- `#豆包输入法`
  - 明确你在说哪种输入方式

首发先不要上这些：

- `#效率工具`
- `#独立开发`
- `#Mac技巧`
- `#iPhone技巧`

原因：

- 你这次想打的是观点和圈层
- 泛标签会把流量打散
- 先把“vibecoding / 豆包输入法 / 大疆”这组词跑出来更重要
